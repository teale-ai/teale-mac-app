# 2026-04-08 — TealeSDK: Third-Party Resource Contribution SDK

## What Was Built

Added **TealeSDK** — a new SPM library that lets third-party app developers monetize their users' idle device resources by contributing to the Teale inference pool. Developers integrate with ~5 lines of code, users opt in via a pre-built UI, and credits flow directly to the developer's Teale wallet.

### New Modules

- **TealeSDK** (`Sources/TealeSDK/`) — 9 files:
  - `TealeContributor.swift` — Main entry point, `@MainActor @Observable`, wires all components
  - `ContributionOptions.swift` — Developer-configurable: RAM limits, schedule, WiFi/power requirements
  - `ContributorState.swift` — State machine + earnings/stats types
  - `ConsentManager.swift` — UserDefaults-backed consent persistence (zero activity before opt-in)
  - `ResourceGovernor.swift` — Enforces RAM limits, uses `vm_statistics64`/`os_proc_available_memory` for real-time pressure
  - `SDKWANBridge.swift` — Connects WANManager to local MLXProvider, handles inference requests, records credits
  - `SDKModelSelector.swift` — Curated 1B-8B model catalog, picks best fit for device
  - `EarningsReporter.swift` — Batches completed work reports for server-side ledger sync
  - `BackgroundScheduler.swift` — iOS BGProcessingTask / macOS background Task

- **TealeSDKUI** (`Sources/TealeSDKUI/`) — 3 files:
  - `TealeContributionView.swift` — Composite view: consent prompt → earnings dashboard
  - `ConsentPromptView.swift` — Opt-in UI with clear explanation bullets
  - `EarningsDashboardView.swift` — Stats, balance, status indicator, opt-out

### Modified Files

- `Package.swift` — Added TealeSDK, TealeSDKUI targets + library product + test target
- `Sources/CreditKit/CreditTypes.swift` — Added `TransactionType.sdkEarning`
- `Sources/CreditKit/CreditAnalytics.swift` — Handle `sdkEarning` in analytics switch
- `Sources/InferencePoolApp/Views/WalletView.swift` — Handle `sdkEarning` in UI switches

### New Migration

- `supabase/migrations/002_sdk_schema.sql` — `sdk_apps`, `sdk_credit_balances`, `credit_reports` tables + RLS policies

## Key Decisions

1. **Wallet-based earnings, no Stripe** — Credits go directly to developer's Teale wallet. No USD conversion needed.
2. **Anonymous auth** — SDK users don't need Teale accounts. Device identity via Ed25519 keypair.
3. **Zero activity before consent** — App Store compliance: no network, no hardware detection until user opts in.
4. **Server-side ledger authoritative** — Local CreditLedger for display; server `credit_reports` table is source of truth.

## Developer API

```swift
import TealeSDK

let teale = TealeContributor(
    appID: "com.example.mygame",
    developerWalletID: "wallet_abc123",
    options: .init(maxRAMContribution: .percent(50), schedule: .afterHours)
)

// In Settings view:
TealeContributionView(contributor: teale)
```

## Local Model Import (added same session)

Added ability to discover and load existing MLX-compatible models from disk, avoiding redundant downloads.

### New Files

- `Sources/ModelManager/LocalModelScanner.swift` — Scans known directories for MLX-format models:
  - `~/.cache/huggingface/hub/` (HuggingFace Hub cache)
  - `~/.cache/lm-studio/models/` (LM Studio)
  - `~/Library/Application Support/Teale/huggingface/` (Teale's own cache)
  - Validates: must have `*.safetensors` + `config.json`
  - Parses `config.json` for model metadata (type, params, architecture)
  - Returns `LocalModelInfo` with path, name, size, source

### Modified Files

- `Sources/MLXInference/MLXProvider.swift` — Added `loadLocalModel(from:descriptor:)` using `LLMModelFactory.shared.loadContainer(from: URL)` (no Downloader needed for local dirs)
- `Sources/InferencePoolApp/AppState.swift` — Added `localModelScanner`, `scannedLocalModels`, `scanLocalModels()`, `loadLocalModel(_:)`
- `Sources/InferencePoolApp/Views/ModelBrowserView.swift` — Added:
  - "Import" menu in footer: "Scan for Local Models" + "Import from Folder..."
  - Local models section with source badges (HF, LMS, TEALE, LOCAL)
  - `.fileImporter` folder picker with MLX compatibility validation

### Key Decisions

- **Ollama models (GGUF) are not MLX-compatible** — only safetensors format works. Scanner only finds MLX-compatible models.
- **No copying** — models load from their original path. No duplication to Teale's cache.
- **Security-scoped resource access** for folder picker (macOS sandboxing).

## iOS On-Device Inference (added same session)

### Changes

- **Package.swift** — Added HardwareProfile, MLXInference, ModelManager, InferenceEngine, CreditKit deps to TealeCompanion
- **CompanionAppState.swift** — Full rewrite: added `MLXProvider`, `ModelManagerService`, `HardwareCapability`, dual-mode `sendMessage()` (local vs remote), model download/load/unload methods, `InferenceMode` enum

- **TealeCompanionApp.swift** — Added "Models" tab with `LocalModelsView`
- **New: LocalModelsView.swift** — Device info, compatible models list, download/load buttons, progress indicators, loaded model status

### How to Test on iPhone 17 Pro

1. Open `Package.swift` in Xcode
2. Select `TealeCompanion` scheme and your iPhone as target
3. Build & run (Cmd+R)
4. Go to **Models** tab → download Gemma 3 4B (2.5 GB, needs 6 GB RAM)
5. Tap **Load into Memory**
6. Switch to **Chat** tab — mode should say "On-device: Gemma 3 4B Instruct"
7. Chat locally, no network needed
