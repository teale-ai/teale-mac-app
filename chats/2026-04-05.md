## What was done

### Phase 1: Local Inference App (from scratch)
- Scaffolded entire Swift package with 7 modules from empty directory
- Built SharedTypes: InferenceProvider protocol, OpenAI-compatible API types, hardware types, model descriptors, engine status, device registration stubs
- Built HardwareProfile: chip/RAM/GPU detection via sysctl + IOKit, thermal/power/activity/network monitors
- Built MLXInference: MLXProvider actor wrapping mlx-swift-lm, custom HFDownloader + HFTokenizerLoader adapters (bridging swift-transformers to MLXLMCommon protocols)

### Phase 2: LAN Cluster Mode
- Created ClusterKit module (13 files)
- Wire protocol: ClusterMessage Codable enum with hello, heartbeat, inference, model sharing message types
- Transport: NWProtocolFramer (4-byte length prefix + JSON), PeerConnection async wrapper, passcode security
- Discovery: BonjourService (NWListener + NWBrowser on _inferencepool._tcp), PeerResolver handshake
- Cluster core: ClusterManager orchestrator, PeerInfo, PeerHealthMonitor (5s/15s/30s), ClusterTopology sorted by capability
- Routing: RequestRouter picks best node, ClusterProvider implements InferenceProvider for cluster delegation
- Model sharing: ModelSharingService with 1MB chunked file transfer
- Refactored InferenceEngineManager to accept `any InferenceProvider` (no longer hardcodes MLXProvider)
- Added ClusterView, PeerCardView, cluster settings section, dashboard cluster summary

## Key decisions
- Used swift-transformers Hub module directly instead of swift-hf-api-mlx adapters (dependency conflict with branch-based mlx-swift-lm)
- In-memory persistence instead of SwiftData (SwiftData macros need Xcode SDK, not available with Command Line Tools)
- Hummingbird 2.x over Vapor (lighter weight) and raw SwiftNIO (too low-level)
- Custom NWConnection protocol over HTTP for inter-node communication (lower latency for heartbeats + streaming)
- MVP cluster uses full-model request routing, not pipeline parallelism (stretch goal)

## Dependency resolution issues encountered
- mlx-swift-examples doesn't export MLXLLM (moved to mlx-swift-lm repo)
- swift-hf-api-mlx / swift-tokenizers-mlx conflict with branch-based mlx-swift-lm dependency
- Solved by writing own Downloader + TokenizerLoader implementations using swift-transformers directly
- NWTXTRecord API differences from documentation (getEntry uses `for:` not `key:`)
- Nested struct memberwise inits are internal across modules — added explicit public inits
